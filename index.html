História da Web

(Voltar)

Não dá para falar da História da Web sem antes falar do surgimento dos primeiros computadores. A história da computação é marcada por varias gerações de computadores:

Um dos primeiros computdores que surgiu foi o Mark I (1944), criado para realizar calculos complexos em pouco espaço de tempo. Em 1946 surge o Colossus, criado para criptoanálises na Alemanha. O colossus foi amplamente utilizado na 2ª Guerra Mundial. Em 1946, já na primeira geração da história da computação surge o ENIAC, primeiro computador automático criado nos Estados Unidos e também foi utilizado na 2ª guerra.

Os computadores da segunda geração tinham como características o uso de transistores, substituindo assim, o uso de válvulas da primeira geração, diminuindo bastante o tamanho desses computadores.

A terceira geração surgiu com a evolução dos cirtcuitos integrados. Os transistores passaram a serem substituídos e o tamanhos dos computadores diminuiram ainda mais.

A quarta geração começa em 1971 com os computadores pessoais, sugimento do mouse e vai até os dias atuais. Há quem diga que já estamos vivendo a era da 5ª geração de computadores, devido ao surgimento de computadores quânticos, IAS, criptomoedas, metaverso, etc.

Já a história da internet começa por volda da década de 50 com a Guerra Fria. Época que alavancou o surgimento de novas tecnologias e do Eniac para realizar cálculos complexos. Nesse mesmo período foi criada a ARPA(Advanced Research Projects Agency) que depois se transformou na DARPA. Foi nesse momento em que surge o protótipo da primeira rede de internet, a Arpanet. Durante meados de 1989, Tim Berners-Lee, desenvolveu a World Wide Web (www), a Rede Mundial de Computadores que conhecemos hoje.

O que são Clients?

(Voltar)

Os clients são dispositivos responsáveis por consumir informações de um determinado lugar. Esses dispositivos podem ser: notebooks, desktops, celulares, tablets, etc. O client é quem envia a requisição HTTP via internet para e servidor e espera que este servidor devolva essa requisição em formato de arquivos: html, CSS e javascript.

A velocidade da internet e o tamanhos dos arquivos pode interfer na experiência do usuário na hora de receber essa requisição. Dessa forma, desenvolvedores frontend, devem desenvolver e implementar soluções que otimizem a experiência do usuário.

O que são Servers

(Voltar)

Os servers são servidores responsáveis pelo envio de informações aos clients. Podem usar internet para se comunicar com client, mas não depende exclusivamente dela, pois poder ser um servidor dentro de uma rede local (computador central) de uma empresa. Normalmente esses computares são mais robustos do que os demais, já que exigem muito processamento. Há vários dispositivos fazendo requisições simultâneas a ele.

Existem diferentes tipos de servidores, dentre eles estão:

  • Proxy:Representa uma ponte entre o usuário e a internet, pois quando um usuário envia uma requisição ao navegador, essa requisição primeiro chegará o Proxy que envia e recebe essa requisição para o servido. Ele é usado para bloqueio de sites malizicosos e també pode ser um servidor de autenticação.
  • Firewall: Este servidor pode pode fazer o memos tipo de segurança do Proxy, a diferença é que ele é voltado para o acesso externo. Todo o tráfego de dados da rede passa por ele.
  • Web Server: Voltado para web este servidor é responsável pelo armazenamento, processamento e entrega dos aqruivos dos sites para os navegadores
  • Database Server: É utilizado apenas para armazenar e gerenciar o banco de dados fonecendo o acesso aos usuários.
  • DNS: Traduz endereços web em Ips númericos levando os usuaŕios aos sites corretos.
  • Hospedagem: São servidores usados para armazenar arquivos de um determinado site deixando-os disponíveis na internet para o usuário acessar.

Estrutura básica de um HTML

(Voltar)

A estrutura de uma página HTML começa com a tag de abertura "<html>" e a segunda tag de fechamento "</html>". Nem todas as tags html possuem fechamento, algumas fecham po si próprias. Dentro das tags html ficam outra duas tags, a "<head></head>" e a tag "<body></body>".

A tag "<head></head>" representa o cabeçalho da página, e dentro dela ficam informações que não são visíveis para o usuário. Essas informações são formadas pelas tags: "<title></tile>" que representa o título que aparece na aba da página, "<link />" que pode ser usada para links css, "<meta>" tag que contém metadados.

A tag "<body></body>" representa o corpo da página, nela contém tudo que é visível para o usuário: parágrafos, listas ordenadas e não ordenadas, imagens, ect. A imagem abaixo mostra como é a estrutura básica de uma página html:

Entendendo a diferença entre listas ordenadas e não ordenadas

(Voltar)

As lista ordenadas são usadas para indicar uma sequência ou numeração. Ela pode ser ordenada por numeros, algarismos romanos ou letras do alfabeto. Para criar uma lista ordenada usamos as tags "<ul></ul>". Dentro dessa tag ficam os itens, representados pelas tags "<li></li>".

As lista ordenadas são usadas para listar items sem se preocupar com a sequência. Para criar uma lista não ordenada usamos as tags "<ol></ol>". Dentro dessa tag ficam os itens, representados pelas tags "<li></li>". Para saber um pouco mais sobre listas clique aqui.
